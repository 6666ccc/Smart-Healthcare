# 患者端纯 AI 医院智能体前端设计

## 目标

将患者端从传统“菜单式互联网医院”改造成以一个医院 AI 智能体为唯一主入口的对话式体验。患者通过自然语言完成健康咨询、就诊分流、预约挂号、查看预约、账单和报告等操作；AI 不具备新增药物、开立处方、修改库存或任何医护端管理权限。

## 设计方向

采用“单智能体工作台”而不是保留传统患者后台导航。桌面端使用三栏结构：左侧为医院身份和会话历史，中间为主对话，右侧为就诊上下文；移动端折叠为单列对话，并把上下文放入可展开面板。视觉使用深靛蓝、冷青绿与暖琥珀的医疗配色，叠加轻量纸张纹理和网格背景，建立可信、安静、有人情味的医院氛围。

## 核心模块

1. **AI 工作台**：欢迎语、推荐问题、消息流、流式回复、输入框、发送状态和医疗免责声明。
2. **患者上下文**：当前患者身份、下一次预约、待缴费金额、最近就诊记录；数据缺失时显示空状态，不阻塞对话。
3. **结构化操作卡片**：预约挂号、查看预约、查看账单、查看报告、联系人工。卡片只允许患者侧已有动作，并在实际变更前要求确认。
4. **会话历史**：本地保存会话标题和消息，支持新建、切换和删除；不得保存敏感的完整身份证号或支付凭据。
5. **安全边界**：前端不渲染医护工具调用、不提供药物增删入口；对急症关键词展示立即就医提示，并保留人工服务入口。

## 数据流

页面加载时并行读取患者基础信息、预约和待缴费摘要；任一摘要接口失败只影响对应卡片。发送消息走现有 `/api/ai/chat` 流式接口；后端返回的工具中断只允许映射到患者白名单操作，未知操作转为安全提示。业务变更使用二次确认，确认后通过现有患者 API 执行并刷新右侧上下文。

## 响应式与可访问性

桌面端在 1180px 以上显示三栏；平板隐藏右侧上下文；移动端使用底部安全区和大触控目标。所有按钮有明确中文标签，输入框支持 Enter 发送、Shift+Enter 换行，颜色不作为唯一状态表达，动画遵守 `prefers-reduced-motion`。

## 验证标准

- `npm run build` 成功。
- 患者进入 `/assistant` 后可在无后端响应时继续浏览欢迎态和上下文空状态。
- 对话页在桌面和移动宽度均无横向溢出。
- 前端代码中不存在患者端创建处方、添加药物或库存管理入口。
