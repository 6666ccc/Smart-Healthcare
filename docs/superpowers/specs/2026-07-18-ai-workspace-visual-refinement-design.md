# AI 工作台视觉精修设计

## 目标

提升患者 AI 工作台首屏的聚焦感、质感与可互动性，保持“医院智能体”作为唯一主入口，并不增加医护端权限或传统后台导航。

## 视觉方向：夜航诊室

保留现有深青侧栏与暖米白主画布，将主对话区调整为由冷青光晕、细颗粒和低对比经纬线组成的安静空间。欢迎态的标志成为缓慢呼吸的“护理灯”，文字与卡片按节奏入场；右侧就诊摘要改为有层次的悬浮信息单元。

## 交互与动画

- 首次进入：品牌、欢迎文字、建议问题和快捷卡片依次上浮，动画总时长低于 700ms。
- 护理灯：低频光晕呼吸，避免持续高频闪烁。
- 建议问题和快捷卡：悬停时轻微上移、边框着色、箭头位移；点击时短暂缩放反馈。
- 输入框：获得焦点时扩大柔和投影，发送按钮在可发送时点亮。
- 摘要卡：悬停露出细色条和行动箭头。
- 尊重 `prefers-reduced-motion`，关闭非必要动画。

## 布局调整

主区域用一条轻量“智能体状态轨迹”连接欢迎区和输入框，减少大面积空白的断裂感；建议问题保持两列，快捷动作提升为带图标底色的横向胶囊。移动端保留单列，不引入影响输入的浮层。

## 验证

- 欢迎态在 1440px 和 390px 宽度无横向溢出。
- 键盘聚焦、悬停与点击状态清晰可见。
- `npm run build` 通过。
