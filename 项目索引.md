# 项目索引 — WenRun 在线医院系统

> **项目简介**: WenRun（问诊）是一套在线医院管理系统，涵盖挂号、问诊、开药、收费、发药、AI 智能辅助等完整门诊流程。项目采用前后端分离 + AI 微服务架构。

---

## 一、顶层目录总览

| 路径 | 类型 | 功能说明 |
|------|------|----------|
| [WenRun/](WenRun/) | 目录 | **Java Spring Boot 主后端服务** — 核心业务 API、OAuth2 认证、数据库交互 |
| [wenrun-AI/](wenrun-AI/) | 目录 | **Python FastAPI AI 微服务** — LangGraph 智能问诊路由、工具调用 |
| [wenrun-react/](wenrun-react/) | 目录 | **React 前端应用** — 患者端 + 医生端双门户 SPA |
| [docs/](docs/) | 目录 | 项目级文档（当前为空） |
| [.claude/](.claude/) | 目录 | Claude Code 本地配置（settings.local.json） |
| [.vscode/](.vscode/) | 目录 | VS Code 工作区配置 |
| [.gitignore](.gitignore) | 文件 | Git 忽略规则 — 覆盖 Python/Java/Node/IDE/环境变量 |

## 六、技术决策要点

1. **OAuth2 自建而非第三方** — Java 端基于 Spring OAuth2 Authorization Server 自建授权服务器，使用数据库存储客户端和令牌，支持 password grant + refresh token 轮换
2. **Python 端通过 OAuth 访问 Java API** — Python AI 服务通过 Client Credentials 获取 Token 后调用 Java 端 API 获取患者数据
3. **AI 采用微服务拆分** — AI 逻辑独立为 Python 服务，Java 端仅做 HTTP 代理转发，便于独立扩展和 LLM 生态兼容
4. **前端双门户响应式** — 同一套 React 代码，通过 PC/Mobile 布局 + 角色路由守卫实现患者端和医生端两套 UI
5. **流式 AI 对话** — 前端通过 SSE（Server-Sent Events）接收 AI 流式响应，Java 端转发 Python 服务的流式输出
